## 롤백 계획 (Rollback Plan)

> 문제 발생 시 되돌리는 방법을 작성해주세요.

- **즉시 롤백**: 해당 커밋을 `git revert <commit-hash>`로 되돌립니다.
- **브랜치 복구**: 머지 전이면 브랜치를 유지한 채 수정 후 다시 PR을 올립니다.
- **영향 범위**: 브랜치 전략, PR 템플릿, 리뷰 체크리스트 문서에 한정됩니다.

---

## 리뷰 체크리스트 (Review Checklist)

> 리뷰어가 확인해야 할 항목입니다.

- [ ] 코드가 브랜치 전략(TBD)을 따르고 있는가?
- [ ] 변경 요약이 실제 코드 변경과 일치하는가?
- [ ] 테스트가 충분히 작성되었는가?
- [ ] 롤백 계획이 현실적으로 실행 가능한가?
- [ ] 불필요한 파일(디버그 코드, 주석 등)이 포함되지 않았는가?
