# 리뷰 체크리스트

PR 리뷰 시 아래 항목을 순서대로 확인합니다.

## 1. 브랜치 전략 준수
- [ ] 브랜치명이 네이밍 규칙을 따르는가? (`feature/`, `fix/`, `docs/`)
- [ ] `main`에 직접 커밋하지 않았는가?
- [ ] 브랜치 수명이 지나치게 길지 않은가?

## 2. PR 설명 완성도
- [ ] 변경 요약이 명확하게 작성되었는가?
- [ ] 테스트 내역이 구체적으로 기술되었는가?
- [ ] 롤백 계획이 포함되어 있는가?

## 3. 코드 품질
- [ ] 불필요한 디버그 코드나 주석이 없는가?
- [ ] 기존 코드 스타일과 일관성이 있는가?
- [ ] 변경 범위가 PR의 목적에 집중되어 있는가?

## 4. 테스트
- [ ] CI 파이프라인이 통과되었는가?
- [ ] 변경 사항에 대한 테스트가 포함되어 있는가?

## 리뷰 코멘트 가이드

| 유형 | 사용 상황 |
|------|-----------|
| `[필수]` | 반드시 수정이 필요한 경우 |
| `[제안]` | 수정하면 좋지만 선택 사항인 경우 |
| `[질문]` | 의도나 구현 방식이 궁금한 경우 |
| `[칭찬]` | 잘 된 부분을 언급할 때 |
