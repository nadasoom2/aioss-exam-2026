# 브랜치 전략 선언

이 프로젝트는 **Trunk-Based Development (TBD)** 전략을 채택합니다.

TBD를 선택한 이유는 문항 4에서 Feature Flag 적용이 요구되며, TBD는 Feature Flag와 결합했을 때 미완성 코드도 `main`에 안전하게 통합할 수 있기 때문입니다.

모든 작업은 `feature/<설명>` 형식의 단명 브랜치에서 진행하며, 완료 즉시 PR을 통해 `main`에 머지하고 브랜치를 삭제합니다.

`main` 브랜치에 직접 push는 금지되며, PR은 반드시 리뷰 체크리스트 확인 후 머지합니다.

CI 파이프라인이 통과된 경우에만 머지를 허용하여 `main` 브랜치의 항상 배포 가능한 상태를 유지합니다.
